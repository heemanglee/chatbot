5. 수동 검증 (BE :8100 / 검증 DB chatbot_verify)

  • 동일 시나리오 재현 시 응답에 <web_context> / </web_context> / <result> / </result> 태그 모두 등장하지 않음.
  • LLM이 자체적으로 "최신 기상 정보 확인" 권유까지 포함하여 응답.

6. Trade-off / 참고

  • OpenAI prompt-cache prefix가 use_web_search boolean에 따라 둘로 분리됨 (with-web / without-web). 동일 사용자가 한 모드를 계속 쓰는 경우가 대부분이므로 cache hit rate 손실은 미미. 안전성 확보를 우선.
  • _RAG_HANDLING_CLAUSE는 같은 패턴 적용 가능하지만 이번 버그 범위에서 제외 (RAG는 항상 활성). 필요 시 후속 PR로 분리.
